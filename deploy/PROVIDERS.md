# Deploy Provider Architecture

## Contract

Every deploy provider is a shell script that reads its configuration from
environment variables and executes these 5 ordered stages:

```
authenticate → prepare → deploy → healthcheck → rollback (on failure)
```

### Required environment variables

| Variable | Description |
|----------|-------------|
| `SSH_HOST` | Target server hostname or IP |
| `SSH_USER` | SSH login user |
| `SSH_PORT` | SSH port (default 22) |
| `REGISTRY` | Container registry hostname |
| `REGISTRY_USER` | Registry login |
| `REGISTRY_PASS` | Registry password/token |
| `IMAGE` | Full `registry/repo:tag` to deploy |
| `COMPOSE_FILE` | Path to docker-compose file on remote |
| `COMPOSE_DIR` | Working directory on remote |
| `SERVICE` | Compose service name |
| `HC_URL` | URL to poll for healthcheck |
| `HC_RETRIES` | Number of healthcheck attempts |
| `HC_INTERVAL` | Seconds between attempts |

## Adding a new provider

### 1. Create the script

```
deploy/providers/provider-<name>.sh
```

Follow the contract above. Copy `provider-docker-ssh.sh` as a template.

```bash
#!/usr/bin/env bash
set -euo pipefail

# Stage 1: Authenticate
# Stage 2: Prepare (capture rollback state)
# Stage 3: Deploy
# Stage 4: Healthcheck
# Stage 5: Rollback hook (on healthcheck failure → exit 1)
```

### 2. Register in the composite action (.github/actions/deploy/action.yml)

Add a new block after the last provider:

```yaml
- name: '[my-provider] Deploy'
  if: inputs.provider == 'my-provider'
  shell: bash
  env:
    SSH_HOST: ${{ inputs.ssh_host }}
    # ... map inputs to env vars
  run: ${{ github.action_path }}/../../scripts/provider-my-provider.sh
```

### 3. Use it in the workflow

```yaml
- uses: ./.github/actions/deploy
  with:
    provider: my-provider      # ← only this line changes
    environment: stage
    image: ${{ needs.docker.outputs.image_tag }}
    # ... other inputs
```

No other workflow changes required.

## Switching the registry provider

The registry is decoupled from the deploy provider.
Change one variable to switch registries across all workflows.

### GitHub Container Registry (default)
```yaml
# In ci-cd.yml env section:
IMAGE_NAME: ghcr.io/${{ github.repository_owner }}/n8go-docs

# In deploy action:
registry_server:   ghcr.io
registry_user:     ${{ github.actor }}
registry_password: ${{ secrets.GITHUB_TOKEN }}
```

### Yandex Container Registry
```yaml
IMAGE_NAME: cr.yandex/${{ secrets.YC_REGISTRY_ID }}/n8go-docs

registry_server:   cr.yandex
registry_user:     iam
# password = current IAM token (generated by yc iam create-token)
registry_password: ${{ steps.yc-token.outputs.token }}
```

### GitLab Container Registry
```yaml
IMAGE_NAME: ${{ env.CI_REGISTRY_IMAGE }}

registry_server:   ${{ CI_REGISTRY }}
registry_user:     ${{ CI_REGISTRY_USER }}
registry_password: ${{ CI_REGISTRY_PASSWORD }}
```

### GitVerse / GitFlic
```yaml
IMAGE_NAME: cr.gitverse.ru/namespace/n8go-docs
# or
IMAGE_NAME: registry.gitflic.ru/namespace/n8go-docs

registry_server:   cr.gitverse.ru
registry_user:     ${{ secrets.REGISTRY_USER }}
registry_password: ${{ secrets.REGISTRY_TOKEN }}
```

## Switching the deploy provider

In `.github/workflows/ci-cd.yml`, change `provider:` in the deploy step:

```yaml
# Before (VPS over SSH):
- uses: ./.github/actions/deploy
  with:
    provider: docker-ssh

# After (Yandex Cloud):
- uses: ./.github/actions/deploy
  with:
    provider: yandex-cloud
    yc_sa_key_json: ${{ secrets.YC_SERVICE_ACCOUNT_KEY_JSON }}
    yc_registry_id: ${{ secrets.YC_REGISTRY_ID }}
```

The rest of the workflow is unchanged.

## Provider: Yandex Cloud (full example)

```yaml
deploy-prod:
  steps:
    - uses: ./.github/actions/deploy
      with:
        provider:      yandex-cloud
        environment:   prod
        image:         cr.yandex/${{ secrets.YC_REGISTRY_ID }}/n8go-docs:${{ github.ref_name }}
        # VM over SSH — same SSH vars as docker-ssh
        ssh_host:      ${{ secrets.PROD_SSH_HOST }}
        ssh_user:      ${{ secrets.PROD_SSH_USER }}
        ssh_port:      ${{ secrets.PROD_SSH_PORT }}
        ssh_key:       ${{ secrets.PROD_SSH_PRIVATE_KEY }}
        # YC-specific
        yc_sa_key_json: ${{ secrets.YC_SERVICE_ACCOUNT_KEY_JSON }}
        yc_registry_id: ${{ secrets.YC_REGISTRY_ID }}
        registry_server:   cr.yandex
        registry_user:     iam
        registry_password: ${{ secrets.GITHUB_TOKEN }}  # placeholder; action generates real token
        compose_file:  docker-compose.prod.yml
        healthcheck_url: http://localhost:8080/health
```
